# CPUX Authoring Tool

## Project Overview
A browser-based CPUX (Client Perceptive UX) authoring tool for the Intentix.lab platform. Helps users design Perceptive Apps through a 5-stage workflow pipeline.

## Architecture (Refactored — Fully Client-Side / Static-Deployable)
- **Auth**: WordPress REST API login (Basic Auth with Application Password OR browser cookie via wp-login.php)
- **Storage**: IndexedDB (via `idb` library) — all projects stored locally in browser
- **LLM**: Configurable API endpoint + key (default: Abacus AI, gpt-5.4-mini). All LLM calls made directly from browser.
- **Session persistence**: Auto-save every 2 seconds to IndexedDB
- **Theme**: Dark mode default, next-themes
- **No server-side dependencies**: No NextAuth, no Prisma, no API routes, no database
- **Static-deployable**: Can be exported and deployed to Vercel, Netlify, GitHub Pages, etc.

## 5-Stage Workflow
1. **Scenario Input**: Text input + example scenarios, LLM streaming analysis
2. **Phrase Discovery**: Phrase CRUD, category assignment (Pulse/DN/I/O), WordPress REST API import
3. **CPUX Synthesis**: Visual IC designer, LLM generation, signal management
4. **Verification**: 8 architectural invariant checks via LLM
5. **Export**: JSON + Markdown export with download/copy

## WordPress API Integration (ISC Phrase API v2)
- Endpoint: `/wp-json/isc/v1/phrases` (configurable in Stage 2)
- Returns `{ phrases: [...] }` with fields: phrase, context, source, sr_type, execution_potential, isc_role, status
- SR types: action_phrase, constraint_phrase, resource_phrase, service_phrase, etc.
- ISC roles: intention, design_node, object, pulse, signal, cpux, etc.

## Key Files

- Settings modal: `app/_components/settings-modal.tsx`
- Dashboard: `app/_components/dashboard-client.tsx`
- Types: `lib/types.ts`
- Example data: `lib/example-scenarios.ts`

## Design Style
- Dark theme with backdrop blur headers
- DM Sans body, Plus Jakarta Sans display, JetBrains Mono monospace
- Primary purple accent, category-specific color coding
- Framer Motion animations
- Card-based layout with lucide-react icons

## Important Notes
- LLM API calls use `response_format: { type: 'json_object' }` - user messages MUST contain word "json"
- All LLM routes use SSE streaming with buffered JSON parsing
